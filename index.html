Problem-Solving Case Studies

Complex defect investigations and high-impact resolutions that prevented critical production failures.

01
Non-Deterministic AI Scheduling Output Producing Invalid Plans
Problem

The AI PSO scheduling engine was producing plans that passed basic functional checks but violated real-world business constraints — over-allocated resources, infeasible task sequencing, and cost estimates outside acceptable bounds. Standard test cases couldn't catch it because no two outputs were identical.

Investigation

Existing test cases validated specific expected outputs — inapplicable for non-deterministic engines. Mapped the domain's hard constraints (resource capacity, task dependencies, cost ceilings) and designed invariant-based checks that held true regardless of the specific schedule produced.

Resolution

Pioneered a constraint-driven validation framework — rules, invariants, feasibility checks, cost bounds, and impact assessments — that validated outputs against business rules rather than exact values. Adopted as the team's standard methodology for all AI output testing.

Zero AI-related production escapes across 90+ sprints. Framework became the R&D standard for non-deterministic validation.

02
Critical Data-Layer Inconsistencies Hidden Beneath Passing UI Tests
Problem

Functional UI tests on the IFS platform were consistently green, yet stakeholder-reported data discrepancies in scheduling outputs persisted across releases. Root cause was unknown and intermittent.

Investigation

Shifted focus below the UI layer. Used Postman for systematic API contract validation and deep SQL queries to verify data integrity across service boundaries. Identified a silent data transformation mismatch at an integration point that corrupted values before they surfaced in the UI.

Resolution

Introduced mandatory API + SQL validation layer as a standard gate in the release checklist. All cross-service data flows now verified at the data layer, not just through UI assertions.

Eliminated the class of silent data bugs. End-to-end data correctness verified across all services and integrations from that sprint forward.

03
Uncontrolled Defect Ageing Blocking Release Pipeline
Problem

Defects were accumulating across sprints with inconsistent severity classifications, unclear ownership, and no enforced resolution timelines. The release pipeline was being blocked by ambiguous defect states and stakeholders lacked confidence in quality reporting.

Investigation

Audited the full open defect backlog. Found that 60%+ of open items lacked reproducible steps, clear severity justification, or assigned resolution accountability — making triage decisions arbitrary and slow.

Resolution

Led 80+ defect triage and quality gate review sessions, establishing reproducibility standards, severity discipline, and clear Dev accountability. Built Jira & Xray dashboards tracking defect ageing, trends, and coverage — producing weekly stakeholder-ready release reports.

Average defect age cut by 35%. Go/no-go decisions became data-driven. Release pipeline unblocked and on-time delivery sustained across 90+ sprints.

04
Flaky AI Test Results Masking Real Solver Regressions
Problem

Test results for the PSO solver were showing pass/fail inconsistency — the team was dismissing real regressions as "just flakiness." Confidence in the suite eroded to the point where failures were routinely ignored, creating invisible risk in every release.

Investigation

Analysed 90 days of test run history. Identified two root causes: timing-dependent race conditions in asynchronous solver callback chains, and non-convergent solver states triggered by specific load parameter combinations that the suite wasn't designed to detect.

Resolution

Separated deterministic from non-deterministic assertions. Introduced statistical confidence intervals for output validation. Added async timeout and retry logic with correct final-state assertions. Rebuilt flaky tests as stable invariant-based checks.

Flake rate reduced from 23% to under 2%. Genuine regressions now reliably surfaced. Team confidence in the test suite fully restored.

05
Cross-Tenant Data Leakage via Unscoped API Endpoints in Multi-Tenant ERP
Problem

New scheduling API endpoints in the IFS multi-tenant architecture were inconsistently enforcing tenant isolation. Under certain HTTP method and payload combinations, data from one tenant's scheduling context was leaking into another — an invisible, intermittent security defect with no visible UI symptom.

Investigation

Built a systematic Postman collection with cross-tenant session tokens and boundary-crossing request patterns. Executed requests across tenant contexts using parameterised variables for tenant scope. Discovered 3 endpoints lacking tenant scope validation in specific method/header combinations — bypassing the expected isolation middleware.

Resolution

Documented the vulnerability pattern and shared it with the Dev and Architecture teams. Collaborated to implement server-side tenant scope enforcement middleware. Created a dedicated security contract test suite covering tenant boundary enforcement — added as a mandatory gate for all new API endpoint sign-offs.

Zero cross-tenant data exposure in production. Security contract suite adopted as a standard gate — catching similar risks across 3 subsequent feature releases before they reached staging.

06
Regression Coverage Collapse from Uncontrolled Test Suite Sprawl
Problem

As the IFS PSO platform grew, the regression suite ballooned to 800+ test cases with 10+ hour execution cycles. Under sprint time pressure, large chunks were being silently skipped. Coverage became fragmented and untracked — the team had no visibility into what was actually being validated each release.

Investigation

Mapped historical defect data against the test suite to identify coverage hot spots. Found that 70%+ of production-bound defects traced back to 18% of the test cases — concentrated in the core solver, constraint engine, and data integration layers. The majority of the suite was validating low-risk stable features that rarely changed.

Resolution

Implemented risk-based test prioritisation using defect heat maps and change-impact analysis. Restructured regression into three tiers: a 90-minute critical-path suite for every sprint, a 4-hour extended suite for major features, and a full overnight run pre-release. Documented coverage tiers in Xray for full stakeholder transparency.

Critical regression cycle cut from 10+ hours to 90 minutes. Full coverage maintained across all tiers. Zero high-severity regressions escaped from the prioritised suite across subsequent releases.
